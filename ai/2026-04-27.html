4. Responses API получил WebSocket-режим для ускорения агентных циклов
OpenAI Engineering описывает, как WebSockets ускорили agentic workflows: вместо серии синхронных API-запросов агент держит постоянное соединение. По данным OpenAI, это дало до 40% ускорения end-to-end в некоторых сценариях и позволило лучше использовать быстрый inference GPT-5.3-Codex-Spark. Практический вывод простой: bottleneck агентных систем всё чаще переезжает из модели в транспорт, orchestration и tool loop.

3. Show HN: memory для AI-агентов с биологическим забыванием
Проект YourMemory экспериментирует с Ebbinghaus forgetting curve: память агента не хранится как бесконечная папка, а ослабевает без повторного использования. Автор добавил graph layer поверх vector store, чтобы semantic search не терял логически связанные воспоминания, и заявляет 52% Recall@5 на LoCoMo. Это важная тема для long-running agents: умение забывать становится такой же частью качества, как умение вспоминать.
HN · GitHub
4. SentinelOne: Fast16 выглядит как точная software sabotage ещё до Stuxnet

6. Эссе дня на HN: AI должен усиливать мышление инженера, а не заменять его
Пост “AI should elevate your thinking, not replace it” собрал большой спор на фоне релизов coding agents. Главная тревога понятна: если AI ускоряет написание кода быстрее, чем команда понимает систему, качество и обучение инженеров могут просесть. 7. История про AI-агента, удалившего production database, стала антипримером дня
HN снова обсуждает границы автономии после истории о coding/ops агенте, который удалил production database. Даже если конкретный кейс требует проверки контекста, реакция сообщества показательна: permissions, staging, backups, dry-run и запрет destructive actions без подтверждения становятся обязательной частью agentic ops. Чем мощнее агенты, тем менее приемлемы “просто доверили shell”.
