OpenAI, ChatGPT и Codex

1. OpenAI выпустила Privacy Filter как open-weight модель для редактирования PII
Новая маленькая модель предназначена для контекстного поиска и маскирования персональных данных в неструктурированном тексте. Ключевая ставка здесь в том, что её можно запускать локально и прогонять большие потоки текста без отправки чувствительных данных на внешний сервер.
OpenAI
2. OpenAI открыла ChatGPT for Clinicians для клинических задач и ресёрча
Компания оформила отдельный clinician-first сценарий вокруг документации, care consult и медицинского поиска, прямо привязав запуск к взрывному росту AI-использования среди врачей. По данным AMA, уже 72% physicians используют AI в клинической практике, так что OpenAI явно пытается закрепиться в одной из самых чувствительных профессиональных вертикалей.
OpenAI
3. Responses API получила WebSocket-режим для ускорения agent loop
OpenAI подробно разобрала, как перешла от цепочки синхронных запросов к persistent connection с кэшем state по `previous_response_id`. На практике это дало до 40% ускорения agentic workflow и позволило Codex реально почувствовать скачок скорости новых coding-моделей вместо того, чтобы упираться в API overhead.
OpenAI
4. Codex CLI 0.123.0 вышел в stable с Bedrock provider и новым `/mcp verbose`
Свежий stable-релиз принёс встроенный `amazon-bedrock` provider, развёрнутую диагностику MCP, улучшения handoff между агентами и обновлённые model metadata с текущим default на `gpt-5.4`. Это один из тех релизов, где в одном пакете одновременно дорабатывают и реальный enterprise connectivity, и повседневный ergonomics для Codex-потока.

AI и Tech радар

1. Google собрала Gemini Enterprise Agent Platform как новый control plane для агентов
На Cloud Next 26 Google свела возможности Vertex AI вместе с новым слоем для интеграции агентов, DevOps, security и governance. Платформа даёт доступ не только к Gemini 3.1 Pro и Lyria 3, но и к моделям Anthropic, то есть Google явно продаёт не одну модель, а целую enterprise-среду для агентных систем.
Google Cloud · Cloud Next 26
2. Google выкатила восьмое поколение TPU как инфраструктуру под agentic era
В Cloud Next компания отдельно вынесла dual-chip подход для самых тяжёлых AI-нагрузок и прямо связала его с новой волной агентных систем. Посыл довольно жёсткий: если все бегут в multi-agent и long-running workload, то инфраструктурное преимущество снова становится не менее важным, чем сами модели.
Google
3. Qwen открыла Qwen3.6-27B как плотную 27B-модель для агентного coding
Alibaba позиционирует её как более практичный dense-вариант без сложностей MoE-маршрутизации, но с very strong coding results. По заявленным benchmark цифрам модель обходит прежний open-source flagship Qwen3.5-397B-A17B на ряде ключевых coding-тестов, что делает релиз важным именно для реально разворачиваемых open models.
Qwen
4. Moonshot продвигает Kimi K2.6 как open-source ставку на long-horizon coding
В техблоге K2.6 описана не как обычный чат-модельный апдейт, а как платформа под длинные инженерные траектории, coding-driven design и agent swarms. Параллельно компания уже раскрыла API pricing с контекстом 262k токенов, что показывает намерение быстро перевести громкий research-style launch в рабочий коммерческий контур.
Kimi blog · Kimi pricing
5. GitHub ужесточает индивидуальные планы Copilot из-за агентной нагрузки
Компания поставила на паузу новые sign-up для Pro, Pro+ и Student, одновременно затянув usage limits и пересобрав доступность моделей. Формулировка в блоге очень прямолинейная: agentic workflow начали съедать заметно больше вычислений, чем старая структура индивидуальных подписок была рассчитана переварить.
GitHub Blog
6. Anthropic выпустила практический разбор trustworthy agents и prompt injection
Новый research-текст уходит от абстрактной safety-риторики к очень прикладной мысли: чем больше у агента инструментов и чем открытее среда, тем больше у атакующего точек входа. Anthropic отдельно подчёркивает, что никакая одна защита не спасает, а реальная agent security складывается только из многослойной обороны и жёсткого контроля прав.
