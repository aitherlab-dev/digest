OpenAI, ChatGPT и Codex

1. OpenAI выпустила GPT-5.5 и завела её в ChatGPT и Codex
Официальный релиз OpenAI описывает GPT-5.5 как модель для сложной работы за компьютером: coding, research, data analysis, documents, spreadsheets и tool use до завершения задачи. В ChatGPT release notes отдельно подчёркнуто, что модель лучше держит контекст больших систем, проверяет себя инструментами и доводит multi-step запросы до результата. Для daily-радарa это главный AI-релиз недели.

2. GPT-5.5 стала доступна в API уже на следующий день после анонса
OpenAI обновила релиз 24 апреля: GPT-5.5 и GPT-5.5 Pro теперь доступны в API, а system card дополнена защитными мерами. Это важно для разработчиков, потому что переход из ChatGPT/Codex в API снимает часть ожидания вокруг production-интеграций. При этом OpenAI подчёркивает отдельные требования безопасности для API-развёртываний.

3. ChatGPT получил Fast answers и отдельный clinical workspace для врачей в США
В ChatGPT release notes за 22 апреля появились Fast answers для быстрых информационных вопросов и ChatGPT for Clinicians для проверенных врачей в США. Клинический workspace обещает evidence review, documentation, medical research, trusted clinical search, citations и CME-кредиты. Это показывает, что OpenAI расширяет не только модельный слой, но и вертикальные рабочие поверхности.
ChatGPT release notes
4. ChatGPT Images 2.0 вышла на все планы ChatGPT
OpenAI начала раскатывать новый генератор изображений в ChatGPT и отдельно добавила images with thinking для paid-планов через Thinking и Pro models. Смысл апдейта в том, что генерация всё меньше выглядит как игрушечный image prompt и всё больше как часть рабочего процесса, где модель может планировать и уточнять визуальный результат. Для презентаций, документов и продуктовых материалов это практичный апдейт.

5. Codex CLI 0.125.0 вышел в stable, а 0.126.0-alpha уже пошёл дальше
Stable-релиз 0.125.0 добавил Unix socket transport для app-server, remote plugin install, marketplace upgrades, перенос permission profiles между TUI/MCP/app-server/API и reasoning-token usage в `codex exec --json`. В тот же день после stable пошли alpha-релизы 0.126.0, последний из которых опубликован 26 апреля. Это не косметика: OpenAI явно донастраивает Codex как платформу вокруг app-server, plugins, permissions и tracing.

6. OpenAI объяснила, почему SWE-bench Verified больше не меряет frontier coding
Пост OpenAI на HN быстро стал одним из главных технических обсуждений дня. Тезис простой: старый benchmark перестал различать верхний край coding-моделей, потому что современные агенты уже упираются не только в патч к GitHub issue, а в более длинные workflow с инструментами, тестами, контекстом и проверкой результата. Это важный сигнал для всей сцены evals: метрика, которая вчера была эталоном, сегодня может стать слишком узкой.

AI и Tech радар

1. DeepSeek выпустила V4 с Pro и Flash версиями и контекстом 1 млн токенов
AP пишет, что DeepSeek V4 включает pro и flash версии, улучшения в reasoning, knowledge и agentic capabilities, а также поддержку китайских Huawei Ascend-чипов. Важная часть релиза - контекстное окно 1 млн токенов, сильный open-source сигнал и попытка снизить зависимость от Nvidia. Независимые проверки ещё нужны, но рынок уже считывает это как новый китайский ответ frontier-моделям США.

2. Anthropic разобрала, почему Claude Code проседал по качеству
Anthropic опубликовала инженерный постмортем по жалобам на Claude Code: default reasoning effort временно снизили, баг с очисткой thinking делал модель забывчивой, а инструкция на меньшую verbosity повредила coding quality. Компания пишет, что изменения откатили или исправили, а подписчикам сбрасывают usage limits. Для рынка agentic IDE это важный прецедент публичного incident-style разбора качества модели и продукта.
Anthropic Engineering
3. Asahi Linux Progress Linux 7.0 стал главным HN-сюжетом дня
Отчёт Asahi Linux собрал верхушку HN, потому что проект остаётся одним из самых сложных и заметных Linux-on-Apple-Silicon направлений. Такие апдейты важны не только пользователям Mac: они показывают, как community reverse engineering, драйверы и графический стек постепенно превращают закрытое железо в более открытое рабочее окружение. В HN это читается как инфраструктурная история, а не просто ещё один distro-апдейт.

4. GnuPG тащит post-quantum crypto в mainline
Анонс GnuPG про landing post-quantum crypto попал в HN как практический security-сигнал, а не абстрактная криптография. Если такие изменения доходят до базовых инструментов шифрования, значит post-quantum migration начинает переходить из research-повестки в реальную supply-chain и операционную подготовку. Особенно важно, что это касается инструмента с огромным историческим следом в open-source инфраструктуре.
