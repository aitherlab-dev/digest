OpenAI, ChatGPT и Anthropic

1. OpenAI выводит модели, Codex и Managed Agents в AWS Bedrock
OpenAI и AWS объявили limited preview: GPT-5.5 и другие OpenAI-возможности приходят в Amazon Bedrock, а Codex можно будет запускать через Bedrock API в CLI, desktop app и VS Code extension. Для enterprise это сильный сигнал после обновления отношений с Microsoft: OpenAI всё активнее разводит продукт, cloud channel и procurement path по нескольким инфраструктурным партнёрам.
OpenAI · Amazon
2. ChatGPT упростил выбор модели прямо в composer
В release notes от 28 апреля OpenAI пишет, что выбор модели переносится в composer, а thinking effort controls переезжают в model picker для Thinking и Pro моделей. Это небольшой UI-релиз, но он важен для everyday UX: модель и глубина reasoning становятся частью ввода запроса, а не отдельным поиском в настройках.
ChatGPT release notes
3. Codex CLI: stable остаётся 0.125.0, alpha дошла до 0.126.0-alpha.12
GitHub releases показывают новый pre-release `0.126.0-alpha.12` от 29 апреля, но его body содержит только короткую строку release без подробных notes. Содержательный stable-релиз остаётся `0.125.0`: Unix socket transport для app-server, remote plugin install, marketplace upgrades, permission profiles, model discovery и reasoning-token usage в `codex exec --json`.
Codex releases · 0.125.0
4. Anthropic запускает Claude for Creative Work с коннекторами к творческим инструментам
Anthropic объявила набор creative connectors для Claude вместе с партнёрами вроде Blender, Autodesk, Adobe, Ableton и Splice. Самый практичный кусок — Claude начинает работать рядом с существующим creative software: помогать с документацией, scripts/plugins, 3D, видео, музыкой и handoff между инструментами, а не только отвечать в чате.

AI и Tech радар

1. GitHub объяснил recent availability incidents ростом agentic workflows
GitHub CTO написал, что после декабря 2025 рост repo creation, PR activity, API usage, automation и large-repository workloads ускорился настолько, что GitHub перешёл от плана 10x capacity к проектированию под 30x текущего масштаба. Это редкий официальный datapoint: агентная разработка уже ломает assumptions инфраструктуры developer platforms.
GitHub Blog · HN
2. GitHub Copilot переходит на usage-based billing с 1 июня
GitHub объявил, что premium request units заменяются GitHub AI Credits, а расход будет считаться по token usage: input, output и cached tokens. Причина прямо названа: Copilot стал агентной платформой с долгими multi-step sessions, и фиксированная подписка больше плохо отражает реальную стоимость inference.
GitHub Blog · HN
3. Wiz раскрыл CVE-2026-3854: RCE в GitHub через обычный git push
Wiz Research описал критическую уязвимость в internal git infrastructure GitHub, которая позволяла authenticated user выполнить команды на backend servers через standard git client. GitHub.com уже mitigated, но GitHub Enterprise Server требует обновления до исправленных версий; это одна из самых заметных security-историй дня для devops-команд.
Wiz · HN
4. Ghostty уходит с GitHub и снова поднимает вопрос platform concentration
Пост Mitchell Hashimoto “Ghostty is leaving GitHub” стал главным HN-тредом дня. Дискуссия уже шире одного terminal emulator: разработчики спорят о зависимости open-source проектов от GitHub, переносимости issues/PR, moderation, outages и том, где должен жить независимый проект.
Ghostty post · HN
5. Warp стал open-source и делает ставку на agent-first contribution workflow
Warp открыл client code под AGPL и описал contribution model, где сообщество помогает ставить задачи и проверять поведение, а agents/Oz делают значительную часть coding/planning/testing. Это важный эксперимент не только для терминала: компания публично тестирует модель open agentic development с issues как source of truth.
Warp · HN
6. Claude Code quality reports получили официальный postmortem, а HN спорит о цене багов
Anthropic выпустила postmortem по ухудшениям Claude Code, Agent SDK и Cowork, заявив, что API/inference layer не были затронуты и проблемы закрыты в v2.1.116. На HN параллельно обсуждают issue про system prompt bug, который мог тратить деньги пользователей и ломать managed agents; важный вывод — агентные продукты требуют observability не хуже backend-сервисов.
Anthropic Engineering · HN
7. Microsoft VibeVoice снова всплыл как сильный open-source voice AI
HN поднял репозиторий VibeVoice: open-source voice AI для long-form audio и speech workflows. Практический интерес для LAB AGENT понятный: локальная или self-hosted обработка встреч, голосовых заметок и длинных записей становится ближе к нормальному workflow, а не отдельному research-проекту.
GitHub · HN
8. Keep Android Open предупреждает о дальнейшей потере контроля над телефоном
Тред “Your phone is about to stop being yours” стал одним из крупнейших на HN. Сюжет вокруг Android/open ecosystem: sideloading, platform rules, app distribution и то, насколько пользователь реально владеет устройством, если производитель и магазин приложений всё сильнее контролируют execution path.
Keep Android Open · HN
9. LocalSend снова в топе как open-source AirDrop для разных платформ
LocalSend — open-source cross-platform file transfer без облака и аккаунта — набрал сильный HN-тред. Это не frontier AI, но полезная инфраструктурная тема: когда устройства, OS и vendor ecosystems плохо дружат между собой, простой локальный transfer tool становится daily productivity инструментом.
GitHub · HN
10. Cua показывает background control macOS-приложений без перехвата курсора
Show HN Cua Driver описывает macOS automation path, где агент кликает, печатает, скроллит и читает native apps, не уводя курсор и focus у пользователя. Это важный кусок computer-use практики: GUI agents должны уметь работать рядом с человеком, а не захватывать всю desktop session.
HN · GitHub
11. OpenAI-пост про GPT-5.4 Pro и задачу Erdős возвращает AI-math в повестку
@OpenAI за последние 24 часа продвигал разбор задачи Erdős, открытой 60 лет и решённой с помощью GPT-5.4 Pro. Даже если это не новый релиз, это сильный research/brand-сигнал: frontier models всё чаще продаются через помощь в математике и scientific discovery, а не только coding и chat.
OpenAI on X
12. OpenAI Symphony появляется как open-source spec для orchestration
На главной OpenAI News зафиксирован engineering-пост “An open-source spec for orchestration: Symphony” от 27 апреля. В связке с AWS Bedrock Managed Agents и Codex app-server это показывает общий тренд: agent orchestration становится отдельным protocol/spec/software layer, а не скрытой частью одного UI.
